# DAG Runtime V2 Stop/Go Synthesis

DAG Runtime V2 stops here.

The round succeeded because one narrow public addition, `SearchAssessment`, was
enough to:

- capture grounded verifier and judge truth
- drive barriered assessment gates
- widen credible paper-family coverage
- preserve the V1 runtime center

## Why V2 stops now

The remaining pressure after Phase 4 does not justify another immediate public
runtime family.

In particular, V2 added **no**:

- async public mode
- new message primitive
- new state primitive
- public reward/search-policy ontology
- DARWIN campaign nouns
- training framework

## Default posture after V2

The default next move is:

- use the assessment-bearing runtime on more recipe and E4 work
- keep optimize composition outside the DAG kernel
- keep DARWIN concerns outside the DAG kernel

## Only proceed beyond V2 if

- multiple families expose the same missing public shape beyond assessments
- private helpers and docs stop being enough
- the missing surface clearly improves coverage without damaging kernel clarity

Until then, DAG Runtime V2 should be treated as frozen.
