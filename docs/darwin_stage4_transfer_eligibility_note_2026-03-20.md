## Note

Promoted does not mean broadly transferable.

The current Stage-4 family program only supports:

- repo_swe topology -> systems as a retained bounded transfer
- systems policy -> scheduling as a bounded comparability failure example

Withheld families remain explicitly non-transferable in the current tranche.

## Practical read

The registry should be read as:

- canonical family status
- explicit transfer bounds
- explicit reasons for disallowance

It should not be read as a broad transfer matrix.
