# DARWIN Stage-3 Component Family Policy V0

Date: 2026-03-19
Status: active for the Stage-3 component-promotion tranche

## Purpose

Define the only promotable Stage-3 component families in the first component-promotion / bounded-transfer tranche.

## In scope

- topology families
- policy-bundle families
- tool-scope families
- evaluator/control families only when bounded and lane-interpretable

## Out of scope

- arbitrary prompt fragments
- broad memory-state artifacts
- untyped mixed bundles
- lane-local hacks presented as reusable components

## Component family versus candidate artifact

A candidate artifact is one concrete bounded run outcome.

A component family is a typed reusable unit inferred from repeated bounded campaign outcomes under matched-budget comparison rules.

## Promotion rule

A component family may be promotion-ready only when:

- repeated bounded campaign evidence exists
- matched-budget comparisons remain valid
- evaluator closure exists
- route and budget telemetry exist
- replay posture is recorded

## Tranche note

This tranche keeps component-family promotion narrow. It does not authorize broad component discovery, broad cross-lane transfer, or archive-as-truth behavior.
