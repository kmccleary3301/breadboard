# Architecture Decision Record (ADR) Template V1

Use this template for architecture decisions that do not alter kernel danger-zone contracts directly.

## Header

- `adr_id`: `ADR-YYYYMMDD-<slug>`
- `status`: `proposed | accepted | superseded`
- `owners`: `[name, ...]`
- `date`: `YYYY-MM-DD`
- `related_acr`: optional `ACR-...`

## Context

What problem is being solved and why now?

## Decision

What is selected and what boundaries are intentionally not changed?

## Alternatives Considered

List rejected alternatives and why.

## Consequences

- expected benefits
- risks / tradeoffs
- rollback trigger

## Validation Plan

- targeted tests
- conformance or parity checks
- evidence artifact paths

## Follow-up

- migration tasks
- owners and due dates
